# SPEC: 반도체 시료 생산주문관리 시스템 — 기능 명세서

> 본 문서는 과제 PDF(`[CRA_AI] Day3_개인과제_반도체시료관리_r1 2.pdf`) Chapter 2 「기능 명세」를 근거로, 각 메뉴 화면의 입력/출력/검증/계산 규칙을 구현 가능한 수준으로 상세화한다. 배경·목표·범위는 [`PRD.md`](./PRD.md), 모듈 분할·계층 구조·PoC 재사용 방식은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 따르며, 본 문서는 그 FR 항목들을 화면 단위로 풀어 쓴 상세 스펙이다. 화면 레이아웃(테두리, 색상 등)은 PDF 예시와 동일할 필요는 없다 — PDF 슬라이드에도 "화면 구성은 자유롭게 결정하여 표기하시기 바랍니다"라고 명시되어 있다.

## 0. 콘솔 공통 규칙

- 모든 메뉴는 번호 입력으로 선택한다. `0`은 항상 "뒤로가기" 또는 "종료"를 의미한다.
- 숫자가 아니거나 목록에 없는 번호를 입력하면 `잘못된 입력입니다. 다시 선택해주세요.` 를 출력하고 같은 메뉴를 다시 표시한다(예외를 던지지 않는다).
- Y/N 확인이 필요한 동작(승인/거절, 예약 확정 등)은 기본값을 두지 않고 `Y` 또는 `N`만 유효 입력으로 처리하며, 그 외 입력 시 재입력을 요구한다.
- 목록 화면에서 페이지네이션이 필요한 경우(시료 10건 초과 등) `[N] 다음페이지` 를 제공한다. 필수 구현은 아니며 전체 출력으로 대체 가능하다.
- 모든 서비스 계층 로직은 콘솔 입출력과 분리되어 있어야 한다(NFR-4). 아래 예시 화면은 View 계층의 출력 형식 참고용이다.

## 1. 메인 메뉴

애플리케이션 시작 시 요약 정보(등록 시료 종수, 총 재고, 전체 주문 건수, 생산라인 대기 건수)와 메뉴를 함께 표시한다.

```
================================================================
              반도체 시료 생산주문관리 시스템
================================================================
 시스템 현황   2026-04-16 09:32:15

 등록 시료  12종     총 재고     2,840 ea
 전체 주문  36건     생산라인    3건 대기
----------------------------------------------------------------
 [1] 시료 관리        [2] 시료 주문
 [3] 주문 승인/거절    [4] 모니터링
 [5] 생산라인 조회     [6] 출고 처리
 [0] 종료
----------------------------------------------------------------
 선택 > _
```

| 항목 | 계산 방법 |
|---|---|
| 등록 시료 종수 | `Sample` 전체 레코드 수 |
| 총 재고 | 모든 `Sample.stock` 합계 |
| 전체 주문 건수 | `Order` 전체 레코드 수 (REJECTED 포함, PDF 예시 기준) |
| 생산라인 대기 건수 | `PRODUCING` 상태 주문 수 (생산 큐에 남아있는 건수) |

잘못된 메뉴 번호 입력 시 안내 메시지 후 메인 메뉴로 복귀한다(FR 5.0, ConsoleMVC 디스패치 패턴).

## 2. 시료 관리 (메뉴 1)

### 2.1 하위 메뉴

```
[1] 시료 관리
------------------------------------------------
 [1] 시료 등록   [2] 시료 목록   [3] 시료 검색   [0] 뒤로
 선택 > _
```

### 2.2 FR-1.1 시료 등록

입력 순서: `name` → `avg_production_time` → `yield_rate` → `stock`. `sample_id`는 자동 채번(`S-001`, `S-002`, ... 등록 순).

검증 규칙 및 실패 시 메시지:

| 필드 | 규칙 | 실패 메시지 |
|---|---|---|
| `avg_production_time` | 숫자, `0` 초과 | `평균 생산시간은 0보다 커야 합니다. 다시 입력해주세요.` |
| `yield_rate` | 숫자, `0` 초과 `1` 이하 | `수율은 0 초과 1 이하 값이어야 합니다. 다시 입력해주세요.` |
| `stock` | 정수, `0` 이상 | `재고는 0 이상의 정수여야 합니다. 다시 입력해주세요.` |

검증 실패 시 등록을 진행하지 않고 해당 필드만 재입력을 요청한다(FR-1.1). `stock`은 사용자가 직접 입력한 값으로 등록된다.

등록 완료 시 출력:
```
등록 완료.
 시료 ID   S-013
 시료명    GaN 에피택셜-6인치
 재고      50 ea
```

### 2.3 FR-1.2 시료 조회

등록된 전체 시료를 표로 표시한다. 컬럼: `ID`, `시료명`, `평균 생산시간`, `수율`, `현재 재고`.

```
등록 시료 목록 (총 12종)
ID      시료명                평균 생산시간    수율    현재 재고
S-001   실리콘 웨이퍼-8인치    0.5 min/ea      0.92    480 ea
S-002   GaN 에피택셜-4인치    0.3 min/ea      0.78    220 ea
S-003   SiC 파워기판-6인치    0.8 min/ea      0.92     30 ea
```

### 2.4 FR-1.3 시료 검색

이름 부분 문자열(대소문자 무관) 검색. 일치 건이 없으면 `일치하는 시료가 없습니다.` 를 출력한다. 결과 형식은 2.3과 동일한 표 형식을 재사용한다.

## 3. 시료 주문 (메뉴 2)

### 3.1 FR-2.1 주문 접수

입력 순서: `sample_id` → `customer_name` → `quantity`.

검증 규칙:

| 필드 | 규칙 | 실패 메시지 |
|---|---|---|
| `sample_id` | 등록된 시료여야 함 | `존재하지 않는 시료 ID입니다. 주문을 생성하지 않았습니다.` (주문 생성 자체를 중단) |
| `quantity` | 1 이상 정수 | `주문 수량은 1 이상의 정수여야 합니다. 다시 입력해주세요.` |

`sample_id` 오류는 재입력을 요구하지 않고 주문 생성을 취소한다(FR-2.1 "오류를 표시하고 주문을 생성하지 않는다"). `quantity` 오류는 재입력을 요구한다.

`order_id`는 `ORD-YYYYMMDD-NNNN` 형식(해당 날짜 내 순번, 4자리 0-padding)으로 자동 채번하고, `created_at`은 ISO8601 UTC로 기록하며, 생성된 주문의 `status`는 `RESERVED`이다.

예시 화면 (PDF 참고):
```
[2] 시료 주문
------------------------------------------------
시료 ID   > S-003
고객명    > 삼성전자 파운드리
주문 수량 > 200

입력 내용 확인
 시료   SiC 파워기판-6인치 (S-003)
 고객   삼성전자 파운드리
 수량   200 ea

[Y] 예약 접수   [N] 취소
선택 > Y

예약 접수 완료.
 주문번호  ORD-20260416-0043
 현재 상태 RESERVED
```

## 4. 주문 승인/거절 (메뉴 3)

### 4.1 FR-3.1 접수된 주문 목록

`RESERVED` 상태 주문만 번호를 매겨 표시한다(주문번호, 고객명, 시료명, 수량, 상태).

```
승인 대기 중인 예약 목록 (RESERVED)
번호   주문번호     고객         시료               수량    상태
[1]    ORD-0041    LG이노텍     산화막 웨이퍼-SiO2  300 ea  RESERVED
[2]    ORD-0042    SK하이닉스   실리콘 웨이퍼-8인치  150 ea  RESERVED
[3]    ORD-0043    삼성전자 파운드리  SiC 파워기판-6인치 200 ea  RESERVED
승인/거절할 번호 > _
```

목록이 비어 있으면 `승인 대기 중인 주문이 없습니다.` 를 출력하고 메인 메뉴로 복귀한다.

### 4.2 FR-3.2 주문 승인

번호 선택 후 승인 여부(`[Y] 승인 [N] 거절`)를 확인한다. 승인 시 재고를 비교해 자동 분기한다.

**재고 충분** (`stock >= quantity`):
```
승인 완료.
 상태 변경   RESERVED → CONFIRMED
 주문번호    ORD-20260416-0042
```
재고는 이 시점에 차감하지 않는다(차감은 출고 시, FR-6.1).

**재고 부족** (`stock < quantity`): 승인 확인 전에 계산 결과를 안내하고 최종 확인을 받는다.

```
재고 확인 중...
 시료      SiC 파워기판-6인치   현재 재고  30 ea
 주문 수량 200 ea               부족분     170 ea  ← 이 수량만 생산

재고 부족. 부족분 170 ea 승인하시겠습니까? (실생산량 206 ea / 165 min)

[Y] 승인   [N] 주문 거절
선택 > Y

승인 완료.
 상태 변경   RESERVED → PRODUCING
 주문번호    ORD-20260416-0043
```

계산식 (FR-3.2 / FR-5.1):

```
부족분     = quantity - stock
실생산량   = ceil(부족분 / yield_rate)
총생산시간 = avg_production_time * 실생산량   (분 단위)
```

승인(`Y`) 시 해당 부족분에 대한 생산이 생산라인 큐에 FIFO로 등록되고 `status`는 `PRODUCING`으로 전환된다. `N` 선택 시 해당 주문은 즉시 `REJECTED`로 전환된다(4.1 목록에서의 `[N] 거절`과 동일한 결과 — 재고 부족 확인 단계에서도 거절을 선택할 수 있다).

### 4.3 FR-3.3 주문 거절

선택한 주문을 즉시 `REJECTED`로 전환한다.
```
거절 처리 완료.
 상태 변경   RESERVED → REJECTED
 주문번호    ORD-20260416-0041
```
`REJECTED` 주문은 이후 모든 모니터링/생산라인 목록·집계에서 완전히 제외된다.

## 5. 모니터링 (메뉴 4)

### 5.1 하위 메뉴
```
[4] 모니터링
------------------------------------------------
 [1] 주문량 확인   [2] 재고량 확인   [0] 뒤로
 선택 > _
```

### 5.2 FR-4.1 주문량 확인

`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE` 4개 상태의 건수를 집계한다. `REJECTED`는 유효한 주문이 아니므로 집계에서 완전히 제외한다.

```
상태별 주문 현황
 RESERVED    3건
 CONFIRMED   8건
 PRODUCING   3건  ← 생산라인 대기
 RELEASE    18건
```

### 5.3 FR-4.2 재고량 확인

시료별 현재 재고와 재고 상태를 표시한다. "대기 수요" = 해당 시료를 참조하는 `RESERVED + PRODUCING + CONFIRMED` 상태 주문의 `quantity` 합.

| 상태 | 판정 기준 |
|---|---|
| 고갈 | `stock <= 0` |
| 부족 | `stock < 대기 수요` (단 `stock > 0`) |
| 여유 | 그 외 (재고가 대기 수요 이상) |

```
재고 현황
시료명                  재고     상태
실리콘 웨이퍼-8인치      480 ea   여유
GaN 에피택셜-4인치       220 ea   여유
SiC 파워기판-6인치        30 ea   부족
산화막 웨이퍼-SiO2         0 ea   고갈
```

## 6. 생산라인 조회 (메뉴 5)

본 시스템은 **단일 생산라인**만 지원하며, FIFO(선입선출)로 큐를 처리한다.

### 6.1 FR-5.4 생산 현황 표시

```
[5] 생산라인 조회   FIFO 방식
------------------------------------------------
생산라인 1개 (단일 라인)   현재 상태: RUNNING

현재 처리 중
 주문번호  ORD-20260416-0038   시료  SiC 파워기판-6인치
 주문량    80 ea   재고 30 ea → 부족 50 ea → 실생산량 54 ea (수율 0.92 / 49 min)
 진행률    [██████░░░░] 72%   완료 예정 09:49
```

진행률/완료 예정 시각 계산 (구현 노트, PRD §5.5 준수):
```
경과시간   = now - 생산_시작_시각
진행률     = min(1.0, 경과시간 / 총생산시간)
완료_예정_시각 = 생산_시작_시각 + 총생산시간
```
시간 소스는 주입 가능해야 한다(NFR-4, DataMonitor DI 패턴).

### 6.2 FR-5.3 생산 완료 처리

큐 맨 앞 항목의 경과시간이 총생산시간 이상이 되면:
1. 해당 시료 `stock += 실생산량`
2. 해당 주문 `status`: `PRODUCING → CONFIRMED`
3. 큐에서 제거, 다음 항목의 생산 시작 시각을 현재 시각으로 설정하고 처리 시작

이 갱신은 실시간 백그라운드 스케줄러 없이, 사용자가 메뉴 4/5/6에 진입할 때(또는 시간 경과 시뮬레이션 트리거) 수행한다.

### 6.3 FR-5.5 대기 큐 표시

FIFO 순서대로 표시. 컬럼: 순번, 주문번호, 시료명, 주문량, 부족분, 실생산량, 예상 완료 시각.

```
대기 중인 주문 (FIFO 순)
순서  주문번호     시료               주문량   부족분   실생산량   예상 완료
1     ORD-0040    산화막 웨이퍼-SiO2   150 ea   150 ea   190 ea    11:43
2     ORD-0043    SiC 파워기판-6인치   200 ea   170 ea   206 ea    14:28
3     ORD-0044    GaN 에피택셜-4인치   300 ea    80 ea   114 ea    15:02

* 부족분 = 주문량 - 재고,  실생산량 = ceil(부족분 / 수율)
* 선입선출(FIFO) 방식으로 처리됩니다.
```
각 대기 항목의 "예상 완료"는 앞선 모든 항목의 총생산시간 누적 + 자신의 총생산시간으로 계산한다(FIFO 순차 처리 가정).

## 7. 출고 처리 (메뉴 6)

### 7.1 FR-6.1 출고 가능 주문 목록

`CONFIRMED` 상태 주문만 번호를 매겨 표시한다(주문번호, 고객명, 시료명, 수량).

```
출고 가능 주문 (CONFIRMED)
번호   주문번호     고객        시료               수량
[1]    ORD-0042    SK하이닉스   실리콘 웨이퍼-8인치  150 ea
[2]    ORD-0035    DB하이텍    포토레지스트-PR7     400 ea
출고할 번호 > _
```
목록이 비어 있으면 `출고 가능한 주문이 없습니다.` 를 출력한다.

### 7.2 FR-6.2 출고 실행

선택 즉시 실행(별도 Y/N 확인 없이 처리, PDF 예시 기준). 실행 시:
1. 해당 시료 `stock -= quantity`
2. 해당 주문 `status`: `CONFIRMED → RELEASE`
3. 결과 표시

```
출고 처리 완료.
 주문번호   ORD-20260416-0042
 출고수량   150 ea
 처리일시   2026-04-16 09:34:02
 상태       CONFIRMED → RELEASE
```

## 8. 계산식 요약 (Single Source of Truth)

| 계산 | 공식 | 사용 위치 |
|---|---|---|
| 부족분 | `quantity - stock` | FR-3.2, FR-5.1 |
| 실생산량 | `ceil(부족분 / yield_rate)` | FR-3.2, FR-5.1, FR-5.5 |
| 총생산시간(분) | `avg_production_time * 실생산량` | FR-3.2, FR-5.1 |
| 대기 수요(시료별) | `sum(quantity)` where status in {RESERVED, PRODUCING, CONFIRMED} | FR-4.2 |
| 재고 상태 | 5.3절 표 참조 | FR-4.2 |
| 생산 진행률 | `min(1.0, (now - 시작시각) / 총생산시간)` | FR-5.4 |

이 5개 공식은 서비스 계층에 단일 구현으로 존재해야 하며, 화면/메뉴 간 중복 계산을 두지 않는다(NFR-7 단일 책임).
