# Phase 12: 통합 검증 및 문서 마감

> 개요/공통 위험·롤백은 [000-overview.md](./000-overview.md) 참고.

## 목표

전체 pytest 스위트와 PRD.md §9 인수 기준을 최종 확인하고, 구현 중 발견된 명세-코드 간 불일치를 문서에 반영해 계획을 마감한다.

## 선행 조건

Phase 0~11 전체 완료.

## 작업 범위

- 전체 `pytest` 스위트 실행 및 통과 확인.
- PRD.md §9 인수 기준 10개 항목을 체크리스트로 하나씩 수동 재현.
- 구현 중 SPEC.md/ARCHITECTURE.md와 실제 코드가 어긋난 부분이 있으면 문서 갱신(코드가 아니라 문서가 잘못 해석된 경우에 한함 — 임의로 명세를 바꾸지 않는다. 명세 자체를 바꿔야 한다고 판단되면 별도로 사용자에게 확인한다).
- 앱 재시작 지속성(NFR-1) 최종 확인.

## 비작업 범위

- 신규 기능 추가 — 없음.
- CI/배포 설정 — 000-overview.md §5 비작업 범위 참고.

## 구현 단계

1. `pytest`(전체, `-q` 또는 기본) 실행, 실패 시 원인 Phase로 돌아가 수정(해당 Phase 커밋에 대한 후속 커밋 또는 Phase별 롤백 후 재작업).
2. PRD.md §9 체크리스트 10개 항목을 순서대로 수동 재현:
   - 시료 등록/조회/검색 + `data/samples.json` 영속화
   - 주문 접수 시 `RESERVED` + `ORD-YYYYMMDD-NNNN` 형식
   - 승인 시 재고 충분/부족 자동 분기(`CONFIRMED`/`PRODUCING`)
   - 거절 시 즉시 `REJECTED` + 이후 전 구간 배제
   - FIFO 큐 계산식(실생산량/총생산시간) 정확성
   - 생산 완료 시 재고 증가 + `PRODUCING → CONFIRMED` 자동 전환
   - 모니터링 상태별 집계(REJECTED 제외) + 재고 상태(여유/부족/고갈)
   - 출고 시 `CONFIRMED → RELEASE` + 재고 차감
   - 앱 재시작 후 데이터 유지
   - pytest 전체 스위트 통과
3. `python main.py` 재시작 테스트: 시료 등록 → 종료 → 재실행 → 등록한 시료가 그대로 조회되는지 확인.
4. 불일치 발견 시 SPEC.md/ARCHITECTURE.md 해당 절만 최소 수정, 수정 이력을 커밋 메시지에 남김.

## 검증 방법

- `pytest` 전체 통과(exit code 0).
- PRD.md §9 체크리스트 10개 항목 모두 체크 완료.
- 재시작 지속성 시나리오 통과.
